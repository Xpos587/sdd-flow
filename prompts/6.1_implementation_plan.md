<DOMAIN_ORGANIZATION_GUIDANCE>

Organize tasks by the domains identified in your Technical Design:

**Typical Domains:**
- `auth` - Authentication, authorization, user management
- `profile` - User profiles, settings, preferences
- `billing` - Payments, subscriptions, invoices
- `notification` - Emails, alerts, messaging
- `admin` - Admin panels, management tools

**Task Categories per Domain:**
- **Frontend Components** - UI components, forms, pages
- **Backend Components** - API endpoints, business logic
- **Shared Components** - DTOs, utilities, common code
- **Database/Infrastructure** - Schema, migrations, setup

**Priority Levels:**
- `High` - Critical path items, blockers
- `Medium` - Important but not blocking
- `Low` - Nice to have, can be deferred

**Dependencies:**
- **Blocks:** What this task prevents from starting
- **Unblocks:** What this task enables
- **Requires:** What must be completed first

</DOMAIN_ORGANIZATION_GUIDANCE>
